// Одними if - else условные операции не ограничиваются, у нас еще есть такая конструкция, как when
// When используется, когда одно значение нужно проверить на множество вариантов или если условия не сложные
// Синтаксис
when(переменная для проверки значения){
	первый вариант значения -> { выполнение кода для этого случая }
	второй ваниант значения -> { выполнение кода для этого случая }
}

//Определим какое число хранится в переменной number
val num = readln().toInt()
when(num){
	1 -> { println("Ваше значение - 1") }
	2..10 -> { println("Ваше значение в промежутке от 2 до 100") }
}

/*
Пример на when():
Светофор: вы вводите цвет светофора в переменную color, в зависимости от введенного
цвета, на экране должны появляться сообщения: Можно идти, Приготовиться, Стоп
Ответ:
*/
println("Введите цвет светофора")
val color = readln()
when(color){
	"Зеленый" -> { println("Можно идти") }
	"Желтый" -> { println("Приготовиться") }
	"Красный" -> { println("Стоп") }
}

// В конструкции when также есть отдельный блок else, который выполняется, когда нет подходящих вариантов.
val login = readln()
when(login) {
	"admin" -> { println("Вы вошли в систему как администратор") }
	"user" -> { println("Вы вошли в систему как пользователь") }
	else -> { println("Извините, вы не можете войти в систему...") }
}

// When конструкция более читаема и используется для множества вариантов одного объекта. 
// А if-else нужно использовать, когда есть всего несколько условий, или эти условия сложные

Задание 1
Напишите программу, которая будет выводить сообщение: "Утро","День","Вечер","Ночь", в зависимости от того какое время вы введете.
Будем принимать утро за 6-11 часов, день за 12-17, вечер за 18-23, а ночь - все остальное.
Эту задачу, конечно, можно решить с if-else, но лучше, чтобы вы попробовали использовать when.
Ответ: */

println("Который сейчас час?(0-23)")
val time = readln().toInt()
when(time){
	in 6..11 -> {
		println("Утро")
	}
	in 12..17 -> {
		println("День")
	}
	in 18..23 -> {
		println("Вечер")
	}
	else -> {
		println("Ночь")
	}
}
