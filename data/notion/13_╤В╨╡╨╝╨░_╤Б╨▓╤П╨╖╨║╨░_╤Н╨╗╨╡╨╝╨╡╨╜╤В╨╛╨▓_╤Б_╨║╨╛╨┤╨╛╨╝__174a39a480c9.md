# Класс для связи управления элементами
Давайте рассмотрим класс, который отвечает за работу с элементами. Так как наш xml-файл с верстой называется activity_main.xml, то класс, отвечающий за него, будет MainActivity. Ключевые моменты: 
функция onCreate() - вызывается при создании нашего экрана и выполняет код, который прописан внутри.
setContentView(R.layout.activity_main) - мы устанавливаем интерфейс activity_main (xml) внутри нашего класса, чтобы работать с самим интерфейсом и его элементами.
Остальные вещи нам пока что не так важны
```kotlin
class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        setContentView(R.layout.activity_main)
        ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main)) { v, insets ->
            val systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars())
            v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom)
            insets
        }
    }
}
```
# findViewById\<\>()
Для того, чтобы найти элементы нашего интерфейса, используется метод findViewById\<Тип Элемента\>(айди элемента), дословно - найти вью-элемент по его айди. Рассмотрим выполнение в коде. 
За макет возьмем интерфейс приложения Телефон. Задача, чтобы при нажатии на кнопку позвонить, у нас появлялось всплывающее уведомление о нажатии этой кнопки.
```kotlin
val bCall = findViewById<Button>(R.id.bCall)
bCall.setOnClickListener(){
    Toast.makeText(this, "click", Toast.LENGTH_SHORT).show()
}
```
Из этого кода видно что мы создаем переменную с таким же названием, как и наша кнопка в макете. Дальше мы ее ищем с помощью findViewById\<Button\>(R.id.bCall). В треугольных скобках мы указали, что это Button, то есть название элемента, как в верстке, в круглых скобках мы вызываем класс R, который хранит ссылки на все ресурсы приложения (макеты, изображения, строки). У класса R мы вызываем свойство id - категория ресурсов внутри этого класса (если нам нужен цвет или что-то другое, то категория будет color и т.п.), ну и у этой категории мы обращаемся к конкретному элементу по его айди. Методом show() мы говорим, что нужно показать уведомление. 
## setOnClickListener()\{\}
Эта функция отвечает за то, что будет происходить при нажатии на элемент. На русском она звучит как установить слушатель нажатия. И в момент, когда наш слушатель заметит нажатие по кнопке, он выполнит код внутри его.
**Toast** — это всплывающее уведомление в Android, которое используется для отображения кратких сообщений пользователю. Оно появляется на экране на короткое время и автоматически исчезает. Toast удобен для отображения временной информации, которая не требует действий от пользователя. Его можно кастомизировать под стиль приложения, при желании.
В коде вызова Toast, мы видим, что вызывается функция makeText - создание самого уведомления, параметры функции:
- **`this`** — контекст - то есть место, в котором будет вызвано уведомление (в данном случае текущая активити, this - этот класс).
- **`"click"`** — текст сообщения.
- **`Toast.LENGTH_SHORT`** — длительность отображения (короткая).
По такому принципу и работают обработки нажатий и других действий.

## Задача 1. Счетчик
Давайте сделаем счетчик нажатий, мы можем как добавлять, так и убавлять его. Дизайн будет таким: по центру располагается цифра 0, внизу есть две кнопки, на которых написано + и - соответственно, они занимают всю ширину с небольшими отступами. Функционал: при нажатии на кнопку - число становится меньше 1, при нажатии на кнопку + число становится больше на 1.
Код верстки: 
```kotlin
<?xml version="1.0" encoding="utf-8"?>
<androidx.constraintlayout.widget.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/main"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <TextView
        android:id="@+id/tvCounter"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="@string/counter"
        android:textSize="40sp"
        android:textColor="#6495ED"
        app:layout_constraintTop_toTopOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintEnd_toEndOf="parent"
        app:layout_constraintBottom_toBottomOf="parent"/>

    <Button
        android:id="@+id/bMinus"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintEnd_toStartOf="@id/bPlus"
        android:layout_margin="8dp"
        android:text="-"
        />

    <Button
        android:id="@+id/bPlus"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        app:layout_constraintEnd_toEndOf="parent"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintStart_toEndOf="@id/bMinus"
        android:layout_margin="8dp"
        android:text="+"
        />

</androidx.constraintlayout.widget.ConstraintLayout>
```
Апдейт к верстке: чтобы писать констрейнты, да и в принципе свойства элементов, быстро, хватает просто написать первую букву каждого слова свойства (не учитывая android или app), например для app:layout_constraintStart_toEndOf хватит написать ste (start to end) и Android Studio само подскажет вам на выбор несколько вариантов, а дальше вам нужно просто найти нужный вариант и нажать на него два раза (или enter).
Код класса: 
```kotlin
class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        setContentView(R.layout.activity_main)
        var counter = 0
        val tvCounter = findViewById<TextView>(R.id.tvCounter)
        val bPlus = findViewById<Button>(R.id.bPlus)
        val bMinus = findViewById<Button>(R.id.bMinus)

        ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main)) { v, insets ->
            val systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars())
            v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom)
            insets
        }
        bPlus.setOnClickListener(){
            //tvCounter.text = (tvCounter.text.toString().toInt() + 1).toString()
            tvCounter.text = (++counter).toString()
        }
        bMinus.setOnClickListener(){
            //tvCounter.text = (tvCounter.text.toString().toInt() - 1).toString()
            tvCounter.text = (--counter).toString()
        }

    }
}
```
Мы изменяем свойство tvCounter.text. 
В первом варианте, мы присваиваем ему свое же значение, но увеличенное или уменьшенное. Чтобы взять это значение текста, нам нужно сначала привести его к строке, так как там другой тип данных CharSequence, а после этого и привести к численному типу и уже проводить с ним операции. 
Во втором варианте, мы создаем переменную счетчика counter и изменяем ее, а ее значение уже присваиваем в tvCounter.text.
Так же, чтобы поместить что-то в TextView, можно использовать метод элемента TextView.setText(текст), но вместо TextView уже писать вашу переменную, например tvCounter.
## Задача 2. Ввод-вывод текста
по центру расположено поле ввода, где написано ввести текст, под ним расположена кнопка Ввод, вверху экрана расположена надпись, но в ней ничего пока нет. Как только пользователь введет что-то в поле и нажмет кнопку Ввод, текст перейдет в надпись, а в поле ничего не останется.
Подумаем, основные элементы: EditText, Button, TextView. При клике текст поля ввода делаем пустой строкой, а введенный сохраняем в отдельную переменную и переносим ее в TextView. <br>Пометка: нужно сделать проверку на то, чтобы поле ввода не было пустым или не состояло только из пробелов. Если же будет наблюдаться такой случай, то показать уведомление о некорректном вводе текста.

Код класса: 
```kotlin
class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        setContentView(R.layout.activity_main)
        val etEnter = findViewById<EditText>(R.id.etEnter)
        val bEnter = findViewById<Button>(R.id.bEnter)
        val tvResult = findViewById<TextView>(R.id.tvResult)

        ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main)) { v, insets ->
            val systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars())
            v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom)
            insets
        }

        bEnter.setOnClickListener(){
            if(etEnter.text.isNotEmpty() && etEnter.text.isNotBlank()){
                val enteredText = etEnter.text.toString()
                tvResult.text = enteredText
                etEnter.text.clear()
            }else{
                Toast.makeText(this, "Некорректно введен текст", Toast.LENGTH_SHORT).show()
            }
        }
    }
}
```

```kotlin
class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        setContentView(R.layout.activity_main)

        val bOne = findViewById<Button>(R.id.bOne)
        val bTwo = findViewById<Button>(R.id.bTwo)
        val bThree = findViewById<Button>(R.id.bThree)
        val etNumber = findViewById<EditText>(R.id.etNumber)
        val bCall = findViewById<Button>(R.id.bCall)
        val bClear = findViewById<Button>(R.id.bClear)
        val tvKeypad = findViewById<TextView>(R.id.tvKeypad)
        val tvRecent = findViewById<TextView>(R.id.tvRecent)
        val tvContacts = findViewById<TextView>(R.id.tvContacts)

        ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main)) { v, insets ->
            val systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars())
            v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom)
            insets
        }

        bOne.setOnClickListener(){
            etNumber.setText(etNumber.text.toString() + bOne.text.toString())
        }
        bTwo.setOnClickListener(){
            etNumber.setText(etNumber.text.toString() + bTwo.text.toString())
        }
        bThree.setOnClickListener(){
            etNumber.setText(etNumber.text.toString() + bThree.text.toString())
        }

        bCall.setOnClickListener(){
            setUpToast("Звоним на ${etNumber.text}")
        }
        bClear.setOnClickListener(){
            etNumber.text.clear()
        }

        tvKeypad.setOnClickListener(){
            setUpToast(tvKeypad.text.toString())
        }
        tvRecent.setOnClickListener(){
            setUpToast(tvRecent.text.toString())
        }
        tvContacts.setOnClickListener(){
            setUpToast(tvContacts.text.toString())
        }
    }

    fun setUpToast(text: String){
        Toast.makeText(this, text, Toast.LENGTH_SHORT).show()
    }

}
```
Как можно увидеть в коде, есть функция setUpToast(), она нужна для того, чтобы не писать каждый раз одно и то же, то есть вызов уведомления, а настроить его и менять только текст внутри. Остальной код можно понять и по проделанной работе.
