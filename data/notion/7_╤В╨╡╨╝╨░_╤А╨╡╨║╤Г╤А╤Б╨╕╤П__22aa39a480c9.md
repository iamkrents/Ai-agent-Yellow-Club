Представим ситуацию:
> Чтобы объяснить, что такое рекурсия,
	нужно сначала объяснить, что такое рекурсия :)))))))))))))
Это и есть идея рекурсии.
## Пример из жизни
- Матрешка внутри матрешки
- Зеркало напротив зеркала
- Папка внутри папки на компьютере
Каждый раз - **то же самое, но чуть проще.**
---
## Условие задачи
Есть функция `countOut`, которая:
- получает число `count`
- выводит `push count`
- уменьшает число на 1 и делает то же самое
- после этого выводит `pop count`
Пример вызова:
```c++
countOut(4);
```
---
## Как мы думаем над задачей
Сначала рассуждаем **словами**, без кода:
> Если мне дали число 4,
	я хочу напечатать 4,
	потом сделать то же самое для 3,
	потом для 2,
	потом для 1.
То есть функция **вызывает саму себя**, но с меньшим числом.
Это и есть **рекурсия**.
---
## Пишем наивный (неправильный) вариант
Попробуем просто написать то, что придумали:
```c++
void countOut(int count)
{
    std::cout << "push " << count << '\n';
    countOut(count - 1);
    std::cout << "pop " << count << '\n';
}

```
Кажется логично, но…
---
## Шаг 3. Что здесь пойдёт не так?
Если вызвать:
```c++
countOut(4);
```
Произойдёт следующее:
```plain text
push 4
push 3
push 2
push 1
push 0
push -1
push -2
push -3
...
```
** Функция никогда не остановится**
Почему?
Потому что:
- мы **уменьшаем число**
- но **никогда не говорим программе, когда остановиться**
---
## Важный вывод
**Любая рекурсия ОБЯЗАНА иметь условие остановки**
Это условие называется:
> базовый случай
---
## Думаем: когда нужно остановиться?
Спросим себя:
> Когда нам больше не нужно вызывать функцию ещё раз?
Ответ очевиден:
- когда `count` стал равен **0**
В этот момент:
- дальше уменьшать уже не нужно
- функция должна **просто завершиться**
---
## Как завершить функцию в C++?
Для этого используется оператор:
```c++
return;
```
---
## Добавляем базовый случай
Теперь перепишем функцию правильно:
```c++
void countOut(int count)
{
    if (count == 0)
        return;   // базовый случай — остановка

    std::cout << "push " << count << '\n';

    countOut(count - 1);

    std::cout << "pop " << count << '\n';
}

```
---
## Шаг 7. Что теперь происходит при вызове `countOut(4)`?
1. `count = 4` → не 0 → работаем дальше
2. `count = 3` → не 0 → работаем дальше
3. `count = 2` → не 0 → работаем дальше
4. `count = 1` → не 0 → работаем дальше
5. `count = 0` → **return**, дальше рекурсии нет
После этого программа начинает возвращаться назад и выводить `pop`.
---
## Результат работы программы
```plain text
push 4
push 3
push 2
push 1
pop 1
pop 2
pop 3
pop 4

```
---
## Главное правило рекурсии (запомнить!)
**Любая рекурсивная функция должна иметь:**
1. **Базовый случай** — когда функция останавливается
2. **Рекурсивный вызов** — вызов самой себя с изменёнными данными
Без базового случая рекурсия =  бесконечная ошибка.

# Рекурсия и цикл `for`: в чём разница?
## Одна и та же задача
**Задача:**
Вывести числа от `1` до `5`.
---
## Вариант 1. Цикл `for` (уже знакомый способ)
```c++
#include <iostream>

int main()
{
    for (int i = 1; i <= 5; i++)
    {
        std::cout << i << " ";
    }
    return 0;
}

```
### Как это работает (простыми словами)
- Завели переменную `i`
- Сказали:
	- начать с `1`
	- пока `i <= 5`
	- каждый раз увеличивать `i` на 1
- Всё происходит **в одном месте**, шаг за шагом
 Цикл — это **повторение действий**.
---
## Вариант 2. То же самое, но с рекурсией
```c++
#include <iostream>

void printNumbers(int n)
{
    if (n == 0)
        return;

    printNumbers(n - 1);
    std::cout << n << " ";
}

int main()
{
    printNumbers(5);
    return 0;
}

```
---
## Как это работает (очень просто)
1. `printNumbers(5)`
	→ вызывает `printNumbers(4)`
2. `printNumbers(4)`
	→ вызывает `printNumbers(3)`
3. `printNumbers(3)`
	→ вызывает `printNumbers(2)`
4. `printNumbers(2)`
	→ вызывает `printNumbers(1)`
5. `printNumbers(1)`
	→ вызывает `printNumbers(0)`
6. `printNumbers(0)`
	→ **останавливаемся (****`return`****)**
После этого числа начинают выводиться обратно:
```plain text
1 2 3 4 5

```
 Рекурсия — это **функция, которая вызывает саму себя**.
---
## Главное сходство
 И цикл, и рекурсия **повторяют действия**
 И там, и там есть **условие остановки**
---
## Главное отличие
### В цикле `for`
```c++
for (int i = 1; i <= 5; i++)

```
- условие остановки — `i <= 5`
- шаг — `i++`
- всё видно сразу
---
### В рекурсии
```c++
if (n == 0)
    return;

printNumbers(n - 1);

```
- условие остановки — `n == 0`
- шаг — `n - 1`
- шаг «спрятан» внутри функции
---
## Очень важное сравнение
 **Цикл** — это как сказать:
> «Сделай это 5 раз»
 **Рекурсия** — это как сказать:
> «Сделай это, а потом попроси себя сделать то же самое, но проще»
---
## Золотое правило для учеников
 **Если можно решить циклом — решай циклом**
 **Рекурсию используй, когда она делает код понятнее**
Но сегодня на занятии мы все задачи будем решать с помощью рекурсии
---
**Пример 1. Простой пример рекурсии**<br>Функция, которая считает от 1 до `n`
```c++
void print(int n) {
    if (n == 0) {
        return; // базовый случай
    }
    print(n - 1); // рекурсивный вызов
    cout << n << " ";
}

int main() {
	print(5); // 1 2 3 4 5
}
```
Как это работает?
```c++
print(5)
 └ print(4)
    └ print(3)
       └ print(2)
          └ print(1)
             └ print(0) → stop
```
## Вопрос, который возникает у всех
Мы **вызываем функцию так**:
```plain text
print(5)
 └ print(4)
    └ print(3)
       └ print(2)
          └ print(1)
             └ print(0) → stop

```
 Почему тогда выводится:
```plain text
1 2 3 4 5

```
а не:
```plain text
5 4 3 2 1

```
---
## Главное правило, которое нужно понять
 **Функция не выполняется целиком сразу.**
 Она может «остановиться на паузе», вызвать другую функцию, а потом продолжить.
---
## Разберём по шагам (очень просто)
###  `print(5)` начинает работать
Код идёт сверху вниз:
```c++
print(4);
cout << 5 << " ";

```
Но сначала нужно **выполнить ****`print(4)`**, поэтому `cout` пока **НЕ выполняется**.
---
###  `print(4)`
Точно так же:
```c++
print(3);
cout << 4 << " ";

```
Опять сначала идём в `print(3)`.
---
###  Так происходит до `print(0)`
Когда вызывается:
```c++
print(0);

```
Срабатывает:
```c++
return;

```
Функция **сразу заканчивается**, ничего не выводя.
---
## И вот тут начинается самое важное
Теперь программа **возвращается назад** и **доделывает то, что не успела**.
---
###  Возврат к `print(1)`
Мы вернулись в строку:
```c++
cout << 1 << " ";

```
 Выводится: `1`
---
###  Возврат к `print(2)`
Теперь выполняется:
```c++
cout << 2 << " ";

```
 Выводится: `2`
---
### И так далее…
```plain text
cout << 3;
cout << 4;
cout << 5;

```
---
## Поэтому итоговый вывод
```plain text
1 2 3 4 5

```
---
## Очень важная идея (обязательно подчеркнуть)
 **Код ДО рекурсивного вызова выполняется при "спуске"**
 **Код ПОСЛЕ рекурсивного вызова выполняется при "возврате"**
---
## Сравнение двух вариантов (очень наглядно)
### Вывод ДО рекурсии
```c++
void print(int n)
{
    if (n == 0) return;

    cout << n << " ";
    print(n - 1);
}

```
Результат:
```plain text
5 4 3 2 1

```
---
### Вывод ПОСЛЕ рекурсии
```c++
void print(int n)
{
    if (n == 0) return;

    print(n - 1);
    cout << n << " ";
}

```
Результат:
```plain text
1 2 3 4 5

```
---
## Как объяснить детям одной фразой
**Рекурсия сначала идёт “вглубь”, а потом “возвращается обратно”**
И именно **в момент возврата** выполняется код после рекурсивного вызова.
---
## Мини-задание для проверки понимания
Что выведет этот код?
```c++
void test(int n)
{
    if (n == 0) return;

    cout << "(" << n << " ";
    test(n - 1);
    cout << n << ") ";
}

```
Пускай напишут и проверят у себя, правильно предположили или нет
## Из чего состоит рекурсия?
Каждая рекурсивная функция **обязана** иметь два элемента:
### Базовый случай
Условие, при котором функция **перестаёт вызывать саму себя**
### Рекурсивный шаг
Вызов функции **с более простой задачей. В**ызов функции самой себя с изменёнными (обычно уменьшенными) параметрами
 Если нет базового случая — будет **бесконечная рекурсия**
<empty-block/>
***Пример вместе***
Задача: Написать функцию factorial(int n), возвращающую n!<br>Математическое определение:<br>n! = n × (n-1) × (n-2) × … × 1<br>0! = 1 (по определению)<br>Рекурсивно:<br>n! = n × (n-1)!<br>1! = 1 и 0! = 1 — базовые случаи.
```c++
#include <iostream>
using namespace std;

int factorial(int n) {  // unsigned long long для больших чисел
    if (n == 0 || n == 1) {     // Базовый случай
        return 1;
    }
    return n * factorial(n - 1);  // Рекурсивный шаг
}

int main() {
    cout << "5! = " << factorial(5) << endl;  // 120
    cout << "10! = " << factorial(10) << endl;
    return 0;
}
```
Рекурсивная функция состоит из двух обязательных частей:
1. **Базовый случай (условие остановки)** — простейший случай, который решается без рекурсии.
2. **Рекурсивный шаг** — вызов функции самой себя с изменёнными (обычно уменьшенными) параметрами.
Разбор на доске:<br>Нарисуйте дерево вызовов для factorial(4):
```c++
factorial(4)
   → 4 * factorial(3)
         → 3 * factorial(2)
               → 2 * factorial(1)
                     → 1 (базовый случай)
               → 2 * 1 = 2
         → 3 * 2 = 6
   → 4 * 6 = 24
```

	Условие:<br>Напишите рекурсивную функцию int sum(int n), возвращающую сумму чисел от 1 до n (n ≥ 0).<br>Пример: sum(5) → 15
	```c++
#include <iostream>
using namespace std;

int sum(int n) {
    if (n <= 0) return 0;          // Базовый случай
    return n + sum(n - 1);         // Рекурсивный шаг
}

int main() {
    cout << sum(5) << endl;   // 15
    cout << sum(10) << endl;  // 55
    return 0;
}
	```

<summary>**Задача 2: Возведение в степень**</summary>
	**Условие:**<br>Напишите рекурсивную функцию long long power(int base, int exp), возвращающую base в степени exp (exp ≥ 0).<br>Пример: power(2, 10) → 1024
	```c++
#include <iostream>
using namespace std;

long long power(int base, int exp) {
    if (exp == 0) return 1;              // Базовый случай: любой^0 = 1
    return base * power(base, exp - 1);  // Рекурсивный шаг
}

int main() {
    cout << power(2, 10) << endl;  // 1024
    cout << power(5, 3) << endl;   // 125
    return 0;
}
	```

	```c++
#include <iostream>

int countDigits(int n) {

}
	```
