# Вводная часть
SFML (сокр. от «Simple and Fast Multimedia Library» = «Простая и быстрая мультимедийная библиотека») — это свободная кроссплатформенная мультимедийная библиотека, написанная на языке C++
Спросить про каждое сложное слово и что оно значит, потом объяснить:
“Библиотека” — это готовый набор кода, который написали другие программисты. Ты можешь не писать всё с нуля, а просто использовать уже готовые функции. Например, не нужно самому придумывать, как вывести картинку на экран — SFML уже умеет это делать.
“Мультимедийная” означает, что библиотека работает с разными видами данных: графикой (картинки, окна), звуком (музыка, эффекты), вводом (клавиатура, мышь).
“Кроссплатформенная” — это значит, что одна и та же программа будет работать на разных операционных системах (например, Windows, Linux, macOS) почти без изменений.
“Свободная” — её можно бесплатно использовать, скачивать и даже изменять под свои задачи.
Если сказать совсем просто: SFML — это инструмент, который сильно упрощает создание игр и программ с графикой и звуком на C++.
С её помощью можно легко и непринужденно создавать любую 2D-графику: начиная от простейших одиночных геометрических фигур (типа треугольника) и заканчивая полноценными играми-платформерами.

# Пишем первый код с помощью библиотеки SFML
Стираем тот код который мы вставляли и пишем ручками свой первый код с помощью данной библиотеки
Наша цель:
- Понять, как работает окно
- Научиться обрабатывать ввод с клавиатуры
- Сделать движение объекта
## Итоговый код который должен получиться
```c++
#include <SFML/Graphics.hpp> // Подключение библиотеки

using namespace sf; // Подключаем пространство имен

int main() // Точка входа в программу
{
    RenderWindow window(VideoMode({ 800, 600 }), "My First Game"); // Создание окна
    RectangleShape player(Vector2f(100.f, 50.f)); // Создание объекта   
    player.setFillColor(Color::Blue); // Цвет объекта
    player.setPosition({ 350.f, 275.f }); // Позиция объекта

    float speed = 1.0f; // Скорость движения (условно)

    while (window.isOpen()) { // Главный цикл программы
        while (const std::optional event = window.pollEvent()) { // Обработка событий
            if (event->is< Event::Closed>()) // Проверка закрытия окна
                window.close(); // Закрываем окно
        }

        if (Keyboard::isKeyPressed(Keyboard::Key::W)) // Управление с клавиатуры
            player.move({ 0.f, -speed });

        if (Keyboard::isKeyPressed(Keyboard::Key::S))
            player.move({ 0.f, speed });

        if (Keyboard::isKeyPressed(Keyboard::Key::A))
            player.move({ -speed, 0.f });

        if (Keyboard::isKeyPressed(Keyboard::Key::D))
            player.move({ speed, 0.f });

        window.clear(); // Очистка экрана
        window.draw(player); // Отрисовка объекта
        window.display(); // Отображение окна
    }

    return 0;
}
```

Сразу после создания окна нам нужно сделать так, чтобы оно не закрылось мгновенно. Для этого мы создаём цикл, который работает, пока окно открыто

```c++
while (window.isOpen()) {}
```
На текущий момент мы не можем закрыть окно через крестик. Для этого нужно написать код, который будет отслеживать события 
## Строка 11: Обработка событий
Начинаем обрабатывать события
Здесь начинается один из самых важных моментов во всей работе с SFML - события. Фактически, вся интерактивность программы строится именно на них. Ранее наши программы просто выполнялись сверху вниз и никак не реагировали на действия пользователя. Теперь же программа “слушает”, что делает пользователь, и реагирует на это
Событие - это любое действие, которое происходит во время работы программы. Например: пользователь нажал кнопку, подвигал мышь, закрыл окно. Всё это превращается в события, которые мы можем обрабатывать в коде.
```c++
while (const std::optional event = window.pollEvent()) {}
```
Мы уже работали с циклами `while`, и знаем, что он выполняется, пока условие внутри - истинно. Здесь смысл такой: пока есть события - мы их обрабатываем. Как только события закончились - выходим из этого цикла и идём дальше.
Теперь разберём, что происходит внутри условия.<br>`window.pollEvent()` - это функция, которая “спрашивает” у окна: есть ли новое событие. Например:
- нажали кнопку
- подвинули мышку
- нажали крестик
Если событие есть - функция возвращает его. Если событий нет - возвращает “ничего”
И вот тут появляется `std::optional`
Ранее мы уже подключали `std`, когда работали со стандартной библиотекой (`std::cout`, `std::string`). Здесь то же самое - `std` означает, что мы используем инструмент из стандартной библиотеки C++
`optional` - это специальный тип, который означает: “значение может быть, а может не быть”
Это удобно, потому что событие не всегда существует. В один момент времени пользователь может ничего не делать - и тогда события просто нет
То есть:
- если событие есть → `optional` содержит значение
- если события нет → `optional` пустой
Теперь смотрим на эту часть: `const std::optional event`
`const` означает, что мы не собираемся изменять это событие - мы только читаем его. Это хорошая практика: если мы не планируем менять данные, лучше пометить их как `const`.
В итоге вся строка читается так: мы вызываем `window.pollEvent()` → если событие есть, оно сохраняется в `event` → цикл выполняется → если событий больше нет, цикл заканчивается
Важно проговорить ключевую идею: окно хранит очередь событий. Мы не получаем сразу всё, а забираем их по одному. Поэтому и используется `while` - чтобы обработать все события, которые произошли с прошлого кадра.
Дальше мы начинаем проверять, что это за события
## Строка 11: Проверка закрытия окна
```c++
if (event->is< Event::Closed>())
    window.close();
```
Здесь мы спрашиваем: это событие - закрытие окна?
`Event::Closed` - это тип события (нажатие на крестик). Если да - вызываем `window.close()`
Важно донести: мы не просто проверяем “закрыто ли окно”, мы реагируем на событие “пользователь хочет закрыть окно”
Это и есть суть событийной модели - программа не угадывает, а реагирует на действия пользователя. Мы постоянно спрашиваем у окна: “что произошло?” → получаем событие → проверяем, какое оно → и решаем, что делать
И именно так работают все интерактивные приложения и игры.
## Строка 8: Создание объекта (грубо говоря игрока)
Создаём прямоугольник, который будет нашим игровым объектом. Мы создаём объект `player` класса `RectangleShape`<br>`RectangleShape` - это класс, который описывает прямоугольник<br>`Vector2f(100.f, 50.f)` — это размер прямоугольника. Здесь используется вектор из двух чисел типа `float`: первое число - ширина, второе - высота.
Таким образом, мы создаём прямоугольник размером 100 на 50 пикселей.
```c++
RectangleShape player(Vector2f(100.f, 50.f));
```
## Строка 16: Отрисовка объекта
Теперь самое главное - чтобы объект появился на экране, его нужно нарисовать внутри цикла
```c++
window.draw(player);
```
Если мы не вызовем `draw`, объект не появится, даже если он создан. Мы передаём `player`, и SFML понимает, что нужно отрисовать именно этот прямоугольник
## Строка 17: Отображение окна
После отрисовки обновляем экран. `display()` показывает всё, что мы нарисовали
```c++
window.display();
```
## Строка 9: Цвет объекта
Задаём цвет нашему объекту
Метод `setFillColor` отвечает за цвет заливки фигуры.
`Color::Blue` - это готовый цвет из библиотеки SFML *(можно и другие цвета)*. Мы используем встроенные цвета, чтобы не задавать их вручную *(**`player.setFillColor(Color(255, 0, 0));`** - если захотят можно через RGB задавать)*
```c++
player.setFillColor(Color::Black);
```
Если вы сейчас запустите код то прямоугольник будет отображаться в левом верхнем углу
## Строка 10: Позиция объекта
Метод `setPosition` задаёт координаты объекта
Мы передаём вектор `{350.f, 275.f}` - это координаты по оси X и Y. X отвечает за движение вправо-влево, Y - вверх-вниз.
По умолчанию используется точка (0, 0) - она находится в левом верхнем углу окна
```c++
player.setPosition({ 350.f, 275.f });
```
## Строка 18: Управление с клавиатуры
Сначала идёт `if`. Мы уже работали с этим оператором - он проверяет условие. Если условие истинно (true), код внутри выполняется. Если нет - пропускается
Дальше смотрим на условие: `Keyboard::isKeyPressed(Keyboard::Key::W)`
`Keyboard` - это класс из SFML, который отвечает за работу с клавиатурой. Мы используем его, чтобы узнать, нажата ли какая-то клавиша
`isKeyPressed` - это функция, которая проверяет: нажата ли конкретная клавиша прямо сейчас. Она проверяет состояние в данный момент, а не “событие нажатия”. То есть если мы держим клавишу - условие будет выполняться каждый кадр
`Keyboard::Key::W` - это сама клавиша, которую мы проверяем. Здесь мы говорим: проверь, нажата ли клавиша W<br><br>В итоге вся строка читается так: если клавиша W сейчас нажата - выполняем действие ниже
Теперь вторая строка: `player.move({ 0.f, -1 });`
`player` - это наш объект (прямоугольник)
`move` - это метод, который сдвигает объект относительно его текущей позиции. То есть мы не задаём новую позицию, а именно “двигаем” его
Внутри передаётся вектор: `{ 0.f, -1 }`. Первое значение - движение по оси X (влево/вправо). Второе значение - движение по оси Y (вверх/вниз). Это не скорость, это **шаг движения за кадр.**
Почему может показаться, что это скорость?
Потому что этот код выполняется **каждый кадр** внутри цикла.
Получается:
- за 1 кадр → сдвиг на -1
- за 60 кадров → сдвиг примерно на -60
И визуально это выглядит как скорость.
```c++
if (Keyboard::isKeyPressed(Keyboard::Key::W))
    player.move({ 0.f, -1 });
```
## Строка 12: Скорость движения (условно)
Создаём переменную `speed`, которая условно отвечает за скорость движения объекта (на самом деле это сдвиг за один кадр)
Мы уже работали с типом `float`, и здесь используем его снова. Значение `0.2f` означает, что объект будет двигаться понемногу каждый кадр
```c++
float speed = 0.2f;
```
Данную переменную мы будем использовать в качестве “скорости отрисовки” и вместо того чтобы писать:
```c++
if (Keyboard::isKeyPressed(Keyboard::Key::W))
    player.move({ 0.f, -1 });
```
Мы будем писать:
```c++
if (Keyboard::isKeyPressed(Keyboard::Key::W))
    player.move({ 0.f, -speed });
```
Это позволит нам менять скорость перемещения нашего прямоугольника изменяя лишь одну переменную, а не в каждом `move`
## Строки 23-30: Управление
Добавим оставшиеся клавиши для управления с клавиатуры
```c++
if (Keyboard::isKeyPressed(Keyboard::Key::W)) // это уже написано
    player.move({ 0.f, -speed }); // это уже написано

if (Keyboard::isKeyPressed(Keyboard::Key::S))
    player.move({ 0.f, speed });

if (Keyboard::isKeyPressed(Keyboard::Key::A))
    player.move({ -speed, 0.f });

if (Keyboard::isKeyPressed(Keyboard::Key::D))
    player.move({ speed, 0.f });
```

Сейчас у нас каждый кадр остается в окне. Поэтому перед тем как что-то рисовать, мы очищаем окно. Это как “стереть доску” перед новым рисунком
```c++
window.clear();
```
