# Frontend разработка
После выполнения работы по HTML + CSS сайт нужно не просто оставить в виде файлов на компьютере, а опубликовать в интернете и оформить результат на сайте WordPress.
На WordPress добавляется отдельная секция с результатом работы:
- название проекта
- кнопка перехода на сайт
- кнопка просмотра кода.
## <span discussion-urls="discussion://353a39a4-80c9-808b-8662-c459fd4c587b/353a39a4-80c9-8024-9e6e-dce6c8c6cd62/361a39a4-80c9-80ee-879e-001c651cfdda">GitHub</span> и GitHub Pages
Раньше мы загружали свои работы на **Яндекс Диск**. Яндекс Диск - это место в интернете, куда можно загрузить файлы. После загрузки можно получить ссылку и отправить её другому человеку. Но у Яндекс Диска есть важное ограничение: он просто хранит файлы. Если мы загрузим туда сайт, то Яндекс Диск не сделает из этого полноценный сайт с адресом. Он просто сохранит файлы.
<callout icon="💡" color="gray_bg">
	Тут мы вспоминаем что необходимо сайту для работы из уроков по html (адрес, сервер, хостинг).
</callout>
Мы уже написали код сайта, но у нас пока нет адреса, хостинга и сервера.
Если просто оставить сайт на компьютере, его увидим только мы. Если просто загрузить папку на Яндекс Диск, другие люди смогут скачать файлы, но это не будет выглядеть как обычный сайт. Поэтому нам нужен сервис, который сможет не только хранить код, но и показывать сайт в интернете.
**Что такое GitHub**
GitHub - это специальный сайт для программистов. На GitHub можно хранить проекты, делиться ими с другими людьми и работать над кодом вместе с командой.
GitHub можно представить как большое облачное хранилище для проектов программистов. Туда можно загрузить все файлы сайта или игры, чтобы они не потерялись и были доступны с любого компьютера.
Также GitHub запоминает изменения в коде. Если мы что-то случайно сломаем, можно вернуться к прошлой версии проекта.
**Что такое GitHub Pages**
GitHub Pages - это функция GitHub, которая умеет превращать папку с кодом в настоящий сайт.
Мы уже знаем, что для работы сайту нужны:
- сервер - компьютер, который хранит сайт;
- хостинг - место, где сайт находится;
- адрес - ссылка, по которой сайт можно открыть.
GitHub Pages помогает получить всё это сразу.
То есть:
- мы загружаем файлы сайта на GitHub;
- GitHub Pages хранит сайт на своём сервере и хостинге;
- сервис автоматически создаёт для сайта адрес;
- после этого сайт можно открыть в браузере как обычную страницу в интернете.
Теперь наш сайт смогут открыть друзья, родители или другие ученики - даже если их нет рядом с нашим компьютером.
Почему это удобно:
- сайт хранится в интернете и не потеряется;
- можно работать над проектом с разных компьютеров;
- GitHub сохраняет историю изменений;
- GitHub Pages бесплатно публикует сайт в интернете;
- можно легко поделиться ссылкой на свою работу.
## Структура проекта
Перед публикацией сайт нужно проверить и привести файлы в правильный вид.
В папке проекта должны быть основные файлы сайта:
```lua
vanya-site/
├── index.html
├── style.css
└── img/
    ├── sun.jpg
    ├── cat.png
    └── favicon.png
```
Главный файл сайта **обязательно должен называться**: `index.html`
Если главный файл будет называться как-то иначе - GitHub Pages может не открыть сайт как главную страницу.
## Подготовка изображений
Перед загрузкой сайта нужно проверить все картинки - **нельзя оставлять картинки только внешними ссылками.**
Плохой вариант:
```html
<img src="https://example.com/image.jpg">
```
Такой вариант может сломаться, если изображение удалят с чужого сайта или ссылка перестанет работать.
Картинки нужно сохранить в папку проекта и привязать относительными ссылками:
- cоздайте папку для изображений
- затем сохраните туда все изображения, которые используются на сайте
- после сохранения изображений измените пути в HTML и CSS
```html
<img src="img/photo.jpg">
```
## Проверка сайта перед загрузкой
Перед загрузкой на GitHub нужно открыть файл `index.html` в браузере и проверить:
1. Сайт открывается
2. Стили CSS работают
3. Все картинки отображаются
4. Кнопки и ссылки работают
5. Нет русских названий файлов с пробелами
6. Главный файл называется именно `index.html`
Лучше использовать названия файлов на английском языке.

## Создание GitHub Pages
После загрузки файлов нужно включить публикацию сайта.
1. Открыть репозиторий
2. Перейти во вкладку **Settings**
3. В меню слева выбрать **Pages**
4. В разделе **Branch** выбрать: main
5. Нажать **Save**
Через некоторое время появится ссылка на сайт. Эту ссылку нужно скопировать и добавить на сайт WordPress.
## Добавление проекта в WordPress
Для HTML+CSS проекта в WordPress нужно создать две кнопки:
1. **Открыть сайт **– эта кнопка ведёт на опубликованный сайт GitHub Pages.
2. **Посмотреть код ** – эта кнопка ведёт на репозиторий GitHub, где лежит код проекта.
К кнопкам привязываем релевантные ссылки.
# Blender
## Проверка модели
Перед тем как загружать модель на сайт, нужно внимательно проверить работу в Blender 
**Важно помнить:** на сайте WordPress модель можно будет крутить мышкой во все стороны. Поэтому модель должна быть аккуратной не только спереди, но и сбоку, сзади, сверху и снизу
**Перед экспортом проверьте:**
- Все объекты стоят на своих местах
- Ничего лишнего не торчит из модели
- Нет объектов, которые случайно вылезают снизу
- Модель выглядит аккуратно со всех сторон
- Нет ненужных кубов, сфер, плоскостей или тестовых объектов
- Все важные детали доделаны
- Цвета и материалы настроены
- Модификаторы применены
- Сцена не перегружена лишними объектами
Если внизу из модели что-то торчит, это нужно исправить. На сайте это будет видно, потому что модель можно повернуть
## **Удаление лишних объектов**
Перед экспортом нужно удалить всё, что не относится к работе.
**Например, можно удалить:**
- лишние кубы
- тестовые объекты
- невидимые детали
- дубли объектов
- слишком мелкие детали, которые почти не видны
- объекты, которые сильно увеличивают вес файла
Это нужно сделать, потому что модель для WordPress должна весить меньше 2 MB
Если модель весит больше 2 MB, её нужно упростить
## Экспорт
После завершения работы модель нужно экспортировать.
Экспорт - это сохранение проекта в специальном формате, который можно открыть в других программах, загрузить на сайт или использовать в игре.
Во время работы Blender сохраняет проект в формате .blend - это внутренний формат Blender. Такой файл хранит все настройки сцены и подходит для редактирования, но его нельзя удобно просматривать на сайте.
Для загрузки на сайт модель нужно сохранить в формате .glb
glb - это удобный формат для 3D-моделей на сайте. В нём могут сохраняться сама модель, материалы и текстуры
**Как экспортировать модель из Blender:**
1. Откройте готовую работу в Blender
2. Проверьте, что модель доделана и выглядит аккуратно
3. Нажмите в верхнем меню: File
4. Выберите: Export
5. Нажмите: .glb/.gltf
## Рендер изображений
Кроме 3D-модели, нужно сделать минимум три фотографии сцены
Это важно, потому что экспортированная 3D-модель на сайте может отображаться без красивого освещения из Blender. А на фотографиях будет видно, как работа выглядит с нормальным светом, камерой и рендером
Можно сделать больше трёх фотографий, если работа большая и интересная. Но минимум должно быть три фотографии 
**Сохраняем фотографии правильно:**
- Фотографии для сайта должны весить меньше: 1 MB
- Если фотография весит больше 1 MB, её нужно уменьшить
**Чтобы уменьшить вес фотографии, можно использовать один из способов:**
- сохранить фото в формате JPG в процессе рендеринга в Blender
- изменить формат через Photoshop
