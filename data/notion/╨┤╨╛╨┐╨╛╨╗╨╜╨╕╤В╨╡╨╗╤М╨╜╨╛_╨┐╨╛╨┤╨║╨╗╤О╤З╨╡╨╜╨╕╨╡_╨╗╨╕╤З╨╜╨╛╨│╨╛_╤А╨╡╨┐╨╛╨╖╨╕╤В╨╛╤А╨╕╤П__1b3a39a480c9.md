# Алгоритм настройки нескольких SSH-ключей для разных пользователей на одном компьютере (GitHub)
### **Шаг 1: Добавить ребенка в коллабораторы к его проекту**
1. Создать аккаунт GitHub для ребенка (если его еще нет).
2. На рабочем аккаунте перейдите в репозиторий ребенка:**Settings** → **Collaborators and Teams** → **Invite a collaborator**.<br>Введите user_name ребенка и отправьте приглашение.
3. Ребенок должен зайти на свой аккаунт и принять приглашение.
### **Шаг 2: Создать SSH-ключ для ребенка**
> Это нужно, чтобы каждый пользователь мог пушить свои изменения без ввода логина и пароля
1. **Откройте терминал и введите:**
	```shell
ssh-keygen -t ed25519 -C "почта_ребенка@gmail.com" -f ~/.ssh/имя_ребенка_фамилия
	```
	🔹 **Объяснение параметров:**
	- `t ed25519` — тип ключа (рекомендуется как более безопасный).
	- `C` — комментарий с почтой, чтобы понимать чей ключ.
	- `f` — путь и имя файла для сохранения ключа.
2. **В папке ****`~/.ssh`**** появятся два файла:**
	- `имя_ребенка_фамилия` — приватный ключ
	- `имя_ребенка_фамилия.pub` — публичный ключ (он-то нам и нужен).
### **Шаг 3: Добавить публичный SSH-ключ в аккаунт ребенка**
1. Перейдите в настройки GitHub ребенка:**Settings** → **SSH and GPG keys** → **New SSH key**.
2. Назовите ключ, например: `<имя_ребенка_фамилия>`.
3. Скопируйте содержимое файла `имя_ребенка_фамилия.pub` и вставьте в поле **Key**.<br>Сохраните.
<details>
<summary>**Шаг 4: Настроить SSH-конфиг для нескольких ключей**</summary>
	> Чтобы не вводить команду ssh-add каждый раз, создадим конфиг.
	1. **Открыть файл конфигурации \~/.ssh/config:**
		Если файл не существует, создайте его.
	2. **Добавьте в файл следующие строки:**
		```shell
# Имя ребенка 1
Host github.com-<username ребенка1>
    HostName github.com
    User git
    IdentityFile ~/.ssh/имя_ребенка_фамилия (это если что приватный ключ)

# Имя ребенка 2
Host github.com-<username ребенка1>
    HostName github.com
    User git
    IdentityFile ~/.ssh/имя_ребенка_фамилия
		```
		🔹 **Объяснение параметров:**
		- `Host` — псевдоним для доступа к разным ключам.
		- `HostName` — реальный адрес GitHub.
		- `IdentityFile` — путь к приватному ключу.
	3. **Сохраните и закройте файл.**
</details>
<details>
<summary>**Шаг 5: Добавить SSH-ключи в ssh-agent (один раз)**</summary>
	> Это нужно, чтобы не вводить пароли при каждом пуше.
	1. Запустите `ssh-agent` (если еще не запущен):
		```shell
eval "$(ssh-agent -s)"
		```
	2. **Добавьте все ключи:**
		```shell
ssh-add ~/.ssh/имя_ребенка_фамилия # приватный ключ ребенка1
ssh-add ~/.ssh/имя_ребенка_фамилия # приватный ключ ребенка2
		```
	3. **Проверьте добавленные ключи:**
		```shell
ssh-add -l
		```
</details>
### **Шаг 6: Настроить Git в проекте ребенка**
> Чтобы каждый проект пушился от имени и почты конкретного ребенка.
В директории проекта ребенка:
1. **Настройте локальный ****`user.name`**** и ****`user.email`****:**
	```shell
git config user.name "username_ребенка"
git config user.email "почта_ребенка@gmail.com"
	```
2. **Проверьте настройки:**
	```shell
git config --list
	```
### **Шаг 7: Настроить удаленный репозиторий для ребенка**
> Чтобы использовать нужный ключ и логин.
1. **Удалите старый ****`remote`**** (если есть):**
	```shell
git remote remove origin
	```
2. **Добавьте новый ****`remote`**** с кастомным ****`Host`****:**
	```shell
git remote add origin git@github.com-<username_ребенка>:yellowclub-by/имя-репозитория.git
	```
3. **Проверьте удаленные репозитории:**
	```shell
git remote -v
	```
**И все должно работать!**

### **Установка и настройка OpenSSH на Windows (PowerShell)**
---
### 🟢 1**. Установить OpenSSH Client (если не установлен)**
> Эта команда скачает и установит клиентскую часть OpenSSH.
```powershell
Add-WindowsCapability -Online -Name OpenSSH.Client~~~~0.0.1.0
```
---
### 🟢 2**. Настроить автоматический запуск ssh-agent**
> ssh-agent хранит SSH-ключи в памяти, чтобы не вводить пароль каждый раз.
1. **Сделать сервис ****`ssh-agent`**** автоматическим:**
	```powershell
Set-Service -Name ssh-agent -StartupType Automatic
	```
2. **Запустить сервис ****`ssh-agent`****:**
	```powershell
Start-Service ssh-agent
	```
3. **Проверить статус сервиса:**
	```powershell
Get-Service ssh-agent
	```
