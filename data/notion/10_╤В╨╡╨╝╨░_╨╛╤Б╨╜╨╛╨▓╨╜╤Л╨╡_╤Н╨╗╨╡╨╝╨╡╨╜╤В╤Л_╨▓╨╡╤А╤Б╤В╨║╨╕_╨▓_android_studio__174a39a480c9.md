# Панель-окно верстки
Панель для верстки экранов.
Для верстки экранов используется язык XML, немного похож на HTML, на котором мы пишем в отдельном файле, либо же просто открываем панель, в котором нужно перетягивать элементы на наш будущий экран и все эти действия все равно будут отражены в файле на языке XML. Чтобы открыть это окно-панель, мы должны посмотреть в наш каталог пакетов (левое меню папок), затем перейти в пакет res, двойное нажатие по папке layout и открываем файл activity_main.xml.

# Элементы верстки и их свойства
Элементы верстки - это элементы которые располагаются на экране мобильного устройства, то есть кнопки, текста, поисковые строки, карты и так далее. То есть если рассмотреть любое приложение на том же телефоне, все экраны этого приложения состоят из множества элементов. Так давайте рассмотрим самые базовые элементы, которые смогут нам понадобиться в разработке нашего итогового приложения Калькулятор.
Чтобы добавлять элементы мы будем использовать Palette - это меню в левой стороне нашей панели верстки. Если нужно будет найти какой-то элемент, то просто используем поиск.

## TextView
TextView - элемент, нужный для отображения текста на экране. Так же нужно запомнить, что текст в последствии нельзя будет изменить, нажав по нему и написав текст с клавиатуры. То есть если мы хотим добавить TextView с текстом "Регистрация", то таким оно и будет.
У каждого элемента есть свои свойства, из-за которых этот элемент выглядит так, как выглядит. В основном свойства начинаются со слова android или app, из-за разных библиотек или пространств имен.  Давайте разберем основные свойства TextView:
- android:id=”@+id/АйдиЭлемента” - обязательное свойство каждого элемента, для определения элемента в коде путем поиска по id. Обычно составляется из первых букв названия элемента и своего названия, подходящего по смыслу, с чем элемент ассоциируется или какое действие он выполняет; то есть id какой-то кнопки с надписью нажми меня будет bPressMe (<span color="green" underline="true">b</span>utton и название Нажми меня <span color="green">PressMe</span>). Значение id мы так же пишем в CamelCase, как и в коде kotlin
- android:layout_width=”100dp” - ширина элемента (по умолчанию - wrap_content, то есть размер, в который точно влезет элемент), единицы dp мы разберем дальше
- android:layout_height=”100dp” - высота элемента (по умолчанию - wrap_content, то есть размер, в который точно влезет элемент)
- android:text="Ваш текст" - Свойство для того, чтобы задать текст для нашего элемента
- android:textSize=”20sp” - размер текста (Обратите внимание, что размер указан в единицах sp, мы затронем это позже), по умолчанию равен 14sp
- android:textColor=”Red” - задаем цвет текста
- android:background=”White” - задаем цвет заднего фона текста
- android:gravity=”center” - выравнивание текста внутри самого элемента (позже мы узнаем о выравнивании внутри контейнера)
- android:fontFamily=”Monospace” - задаем шрифт текста
- android:textStyle=”italic” - свойство стиля текста (жирный, курсив)
Чтобы изменять свойства, пока что мы просто нажимаем на элемента, а затем на иконку поиска возле Attributes в правом верхнем углу и ищем нужное нам свойство.

Свойства EditText: 
- android:id
- android:hint=”Введите логин” - подсказка для пользователя, что он мог догадаться, какую информацию ему вводить, отображается серым цветом

- android:layout_width=”wrap_content” 
- android:layout_height=”wrap_content”

# Размеры sp и dp
Немного информации про sp и dp. Это специальные единицы измерения, которые расчитываются исходя из размера экрана, для того, чтобы на любом устройстве экраны приложения и элементы выглядели одинаково. Sp используется у размера текста, а dp в остальных случаях.

У полей ввода есть подсказки: email, пароль; соответственно, должны быть выставлены правильные типы ввода. Ширина полей ввода и кнопки - 280dp. Цвет кнопки - зеленый, цвет текста подписей и полей ввода - черный. Надпись с вопросом будет голубого цвета, а размер по умолчанию, стиль текста - курсив.
