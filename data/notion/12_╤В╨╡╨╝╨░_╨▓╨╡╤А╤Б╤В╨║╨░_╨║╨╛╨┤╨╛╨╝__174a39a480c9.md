# Верстка с помощью кода
До этого мы верстали экраны, перетаскивая элементы, теперь же будем для верстки использовать код - язык XML. 
Это облегчает верстку: не нужно искать свойства, не нужно оттягивать элементы. Достаточно описать эти действия несколькими строками кода.

## Разбор кода элемента
В окне слева у нас отобразится код базового экрана.
```kotlin
<androidx.constraintlayout.widget.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/main"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello World!"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintEnd_toEndOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintTop_toTopOf="parent" />

</androidx.constraintlayout.widget.ConstraintLayout>
```
Видим уже знакомые нам TextView, ConstraintLayout.  
<callout icon="☑️" color="gray_bg">
	Попросить разобрать свойства TextView, если будут трудности, то вместе.
</callout>
Давайте разберем TextView.
`android:layout_width="wrap_content"` - ширина подписи такая, что должна вмещать весь контент(текст)
`android:layout_height="wrap_content"` - высота подписи такая, что должна вмещать весь контент(текст)
`android:text="Hello World!"` - текст внутри подписи - Hello, World!
`app:layout_constraintBottom_toBottomOf="parent"` - констрэйнт или ограничение, которое говорит о том, что нижний край нашего элемента относится к нижнему краю parent, то есть родителю, коим в данном случае выступает контейнер
`app:layout_constraintStart_toStartOf="parent"` - похожее свойство, которое говорит о том, что начало подписи относится к началу родителя
## Добавление элементов
Давайте добавим новый элемент с помощью кода. Поместим текстовое поле (поле ввода) внутри контейнера под надписью. 
<callout icon="🧑‍🎓" color="gray_bg">
	Какое наименование носит элемент поле для ввода?
	<callout icon="🙋‍♂️" color="yellow_bg">
		 EditText
	</callout>
</callout>
Чтобы добавить элемент необходимо написать его название, поместив в треугольные скобки:
`<EditText/>`
Пишем `<EditText` и нажимаем enter. У нас появится элемент с базовыми свойствами. В конце закрываем наш тег - `/>`. 
## Свойства элементов
Чтобы добавить свойства, нам не нужно начинать их со слов android или app. Мы пишем название свойства, после выбираем из подсказок нужный нам вариант и нажимаем enter.
<callout icon="🔗" color="gray_bg">
	<mention-page url="https://app.notion.com/p/174a39a480c98020b390c2919551dd79"/> 
</callout>
### Размер
Свойство высоты и ширины будет добавлено автоматически. Нам будет необходимо только выбрать значение.
`wrap_content` или `match_parent`  - эти свойства мы знаем.
### Привязка
Так как элемент должен быть под надписью, то это значит, что его верх должен быть закреплен к нижнему краю надписи. Также не стоит забыть про закрепления по горизонтали, соответственно, начало элемента к началу родителя, конец к концу родителя.
<callout icon="💡" color="gray_bg">
	Не забываем про то, что закрепление к элементу осуществляется по его id, то есть нужно его еще добавить.
</callout>
Вспоминаем правила.
`layout_constraintTop_toTopOf` - верхняя граница позиционируется относительно верхней границы другого элемента
`layout_constraintTop_toBottomOf` - верхняя граница позиционируется относительно нижней границы другого элемента
`layout_constraintBottom_toBottomOf` - нижняя граница позиционируется относительно нижней границы другого элемента
`layout_constraintBottom_toTopOf` - нижняя граница позиционируется относительно верхней границы другого элемента
`layout_constraintStart_toEndOf` - элемент начинается там, где завершается другой элемент
`layout_constraintStart_toStartOf` - элемент начинается там, где начинается другой элемент
`layout_constraintEnd_toStartOf` - элемент завершается там, где начинается другой элемент
`layout_constraintEnd_toEndOf` - элемент завершается там, где завершается другой
Также добавляем отступы. Для этого используем следующие свойства:
`marginTop`
`marginStart`
`marginBottom`
`marginEnd`
`padding` - внутренние отступы (разбирали в уроке с элементами)
### Внешний вид
Селаем задний фон элемента серым.
`android:background="#c5c5c5"`
`background` - свойство для изменения цвета фона элемента
`#c5c5c5` - код серого цвета в системе HEX.
<callout icon="💡" color="gray_bg">
	У некоторых элементов это может быть свойство `backgroundColor`.
</callout>
В данном случае при смене цвета поля ввода нам нужно сделать внутренние отступы, так как сам элемент уменьшается.
Добавим подсказку для пользователя, которая при нажатии не будет мешать вводить текст.
`android:hint=”Введите какой-то текст”` - 
Конечный код должен выглядеть так.
```kotlin
    <EditText
        android:id="@+id/etTest"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:hint="Введите какой-то текст"
        android:background="#c5c5c5"
        android:padding="10dp"
        app:layout_constraintEnd_toEndOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintTop_toBottomOf="@id/tvText" />
```
