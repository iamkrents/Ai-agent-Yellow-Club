В обычном (традиционном) программировании программа — это просто набор команд. Мы создаём данные (например, переменные или структуры), а потом пишем функции, которые с этими данными работают. При этом данные и функции существуют как будто отдельно друг от друга, и программисту приходится самому следить, какие функции к каким данным относятся.
Например, можно написать функцию `driveTo(you, work);` — «поехать из точки you в точку work». Но из такого кода не очень понятно, кто именно выполняет действие. Мы просто передаём данные в функцию.
Теперь представим реальный мир. Вокруг нас всё состоит из объектов: человек, машина, телефон, книга. У каждого объекта есть характеристики — например, цвет, размер, имя. Это называется свойствами. И у каждого объекта есть действия, которые он может выполнять — это называется поведением. Например, человек может идти, говорить, ехать куда-то.
В объектно-ориентированном программировании мы стараемся программировать так же, как устроен реальный мир. Мы объединяем свойства и поведение в одну сущность — объект.
То есть объект — это просто «штука», у которой есть данные (свойства) и действия (методы), и они всегда идут вместе.
Теперь тот же пример можно записать так: `you.driveTo(work);`. Здесь уже понятно, что именно объект `you` (то есть «ты») выполняет действие `driveTo` (поехать). Код читается почти как обычное предложение.
Такой подход делает программу более понятной: мы думаем не в терминах «какую функцию вызвать», а в терминах «какой объект что делает». Это сильно упрощает работу, особенно в больших проектах.
Кроме того, объекты удобно переиспользовать. Если мы один раз описали, как работает, например, класс «Машина», мы можем создавать сколько угодно машин и использовать их в разных местах программы.
Важно понимать, что объектно-ориентированное программирование не заменяет обычное программирование. Это просто другой способ организации кода, который помогает лучше справляться со сложными задачами.
## Пример
Представим, что мы хотим описать человека в программе.
В обычном подходе мы могли бы хранить отдельно имя, возраст, а действия делать через функции. Но в ООП мы объединяем всё в один объект.
В C++ для этого используется **класс** — это как «чертёж» объекта.
```c++
#include <iostream>
using namespace std;

class Person {
public:
    string name;
    int age;

    void sayHello() {
        cout << "Привет, меня зовут " << name << endl;
    }

    void driveTo(string place) {
        cout << name << " едет в " << place << endl;
    }
};
```
Здесь мы описали класс `Person` (человек).
У него есть свойства:
- `name` — имя
- `age` — возраст
И есть поведение:
- `sayHello()` — поздороваться
- `driveTo()` — поехать куда-то
Теперь создадим конкретный объект:
```c++
int main() {
    Person you;

    you.name = "Алекс";
    you.age = 16;

    you.sayHello();
    you.driveTo("школа");

    return 0;
}
```
Здесь происходит самое важное:
`Person you;` — мы создали объект (конкретного человека) на основе класса.
Дальше мы задаём ему свойства:
`you.name = "Алекс";`
И вызываем его поведение:
`you.sayHello();`
`you.driveTo("школа");`
---
Обратите внимание, как это читается:
`you.driveTo("школа");` — «ты едешь в школу»
Это и есть суть ООП — код становится похож на реальную жизнь и обычную речь.
---
Если совсем просто:
- класс — это шаблон (например, «человек вообще»)
- объект — это конкретный экземпляр (например, «Алекс, 16 лет»)
## Модификаторы доступа
В C++ у каждого свойства и метода можно указать, **кто имеет к ним доступ**. Это называется **модификаторы доступа**.
Проще говоря, это способ сказать: «этим можно пользоваться всем» или «это только для внутреннего использования».
В C++ есть три основных модификатора: `public`, `private` и `protected`.
Начнём с самого понятного — `public`.
Если что-то объявлено как `public`, к нему можно обращаться откуда угодно.
```c++
class Person {
public:
    string name;
    int age;

    void sayHello() {
        cout << "Привет, меня зовут " << name << endl;
    }

    void driveTo(string place) {
        cout << name << " едет в " << place << endl;
    }
};

int main() {
	Person p;
	p.name = "Алекс";
	p.age = -100;  // 🤨 так тоже можно!
}
```
И вот здесь появляется проблема — мы никак не защищаем данные. Программа позволяет записать ерунду, например отрицательный возраст.
Теперь покажем, как это исправить с помощью `private`.
Мы просто «прячем» возраст внутрь класса:
```c++
class Person {
public:
    string name;

    void sayHello() {
        cout << "Привет, меня зовут " << name << endl;
    }

    void driveTo(string place) {
        cout << name << " едет в " << place << endl;
    }

    void setAge(int a) {
        if (a > 0) {
            age = a;
        }
    }

    void showAge() {
        cout << "Возраст: " << age << endl;
    }

private:
    int age;
};

int main() {
	// Теперь снаружи так сделать уже нельзя:
	Person p;
	p.age = 16; // ❌ ошибка
	
	// Но можно через методы:
	
	p.setAge(16);  // ✅ правильно
	p.showAge();
}
```
Важно проговорить с ними идею: мы не просто «запрещаем доступ», мы **контролируем, как данные изменяются**.
Например, теперь невозможно случайно задать отрицательный возраст.
Можно прямо задать им вопрос: «Как думаете, что ещё в этом классе стоит сделать `private`?»
И подвести к мысли, что `name` тоже можно спрятать и сделать, например, `setName()`.
## Инкапсуляция
Когда мы сделали `age` приватным, мы на самом деле уже использовали важный принцип ООП — **инкапсуляцию**.
Инкапсуляция — это когда мы **прячем внутренние данные объекта и разрешаем работать с ними только через специальные методы**.
Проще говоря: объект сам решает, что с ним можно делать, а что нельзя.
```c++
class Person {
public:
    void setName(string n) {
        if (n != "") {
            name = n;
        }
    }

    void setAge(int a) {
        if (a > 0) {
            age = a;
        }
    }

    void sayHello() {
        cout << "Привет, меня зовут " << name << endl;
    }

    void driveTo(string place) {
        cout << name << " едет в " << place << endl;
    }

    void showAge() {
        cout << "Возраст: " << age << endl;
    }

private:
    string name;
    int age;
};

int main() {
	Person p;

	// p.age = 10;   ❌ нельзя
	// p.name = "";  ❌ нельзя
	
	p.setName("Алекс"); // ✅ правильно
	p.setAge(16);       // ✅ правильно
}
```
Вот это и есть инкапсуляция. Объект как будто говорит: «Ты не можешь трогать мои данные напрямую. Если хочешь что-то изменить — используй мои методы».
Хорошая жизненная аналогия — банкомат. Ты не можешь открыть его и взять деньги напрямую. Ты используешь кнопки (интерфейс), и банкомат сам решает, можно ли выдать деньги.
Точно так же и объект:
- данные спрятаны внутри
- снаружи доступны только «кнопки» (методы)
**Почему это важно:**
- код становится безопаснее — нельзя случайно сломать данные
- код становится понятнее — видно, как правильно работать с объектом
- проще изменять программу — можно менять внутреннюю реализацию, не ломая остальной код
## Наследование
Иногда нам нужно создать новый класс, который почти такой же, как уже существующий, но с небольшими изменениями или дополнениями.
Например, у нас уже есть класс `Person`. Но мы хотим сделать, например, студента. Студент — это тот же человек, только у него есть ещё что-то своё (например, университет).
Чтобы не копировать весь код заново, в ООП используется **наследование**.
Наследование — это когда один класс «наследует» свойства и методы другого класса.
Допустим, мы хотим сделать класс `Student` (студент).
```c++
class Student : publicPerson {
	public:
	void study() {
		cout<<"Учится..."<<endl;
	}
};
```
Вот эта часть: `class Student : public Person` означает: «класс Student наследуется от класса Person»
Теперь создадим объект студента:
```c++
int main() {
	Student s;
	
	s.setName("Алекс");
	s.setAge(16);
	
	s.sayHello();// из Person
	s.study();// своё
	
	return 0;
}
```
И происходит магия ✨
Объект `Student` уже умеет:
- задавать имя (`setName`)
- задавать возраст (`setAge`)
- здороваться (`sayHello`)
- ехать куда-то (`driveTo`)
И плюс у него есть своё поведение — `study()`. 
Важно объяснить им одну вещь:`Student` — это **тот же Person, но с дополнительными возможностями**.
Теперь важный момент, который они, скорее всего, заметят.
В нашем классе `Person` поля `name` и `age` — `private`.
Это значит, что `Student` не может к ним напрямую обратиться.
Например, так не сработает:
```c++
class Student :public Person {
	public:
	void printName() {
		cout<<name;// ❌ ошибка
	}
};
```
Почему? Потому что `private` — это полностью закрыто даже для наследников.
И вот тут можно аккуратно ввести `protected`.
Если мы хотим, чтобы наследники имели доступ, делаем так:
```c++
class Person {
public:
    void setName(string n) {
        if (n != "") {
            name = n;
        }
    }

    void setAge(int a) {
        if (a > 0) {
            age = a;
        }
    }

    void sayHello() {
        cout << "Привет, меня зовут " << name << endl;
    }

protected:
    string name;
    int age;
};
```
<br>Теперь `Student` сможет использовать `name` напрямую.
```c++
class Student : public Person {
public:
    void introduce() {
        cout << "Я студент, меня зовут " << name << endl;
    }
};
```
И итог, который важно проговорить:
наследование нужно, чтобы<br>- не копировать код<br>- расширять уже существующие классы<br>- строить иерархию (человек → студент → отличник и т.д.)
Очень простая формула для них: `Student` — это `Person` + что-то ещё
## Полиморфизм
Полиморфизм — это сложное слово, но идея простая: **один и тот же метод может работать по-разному у разных объектов.**
У нас уже есть базовый класс `Person`:
```c++
class Person {
public:
    void setName(string n) {
        if (n != "") {
            name = n;
        }
    }

    void setAge(int a) {
        if (a > 0) {
            age = a;
        }
    }

    void sayHello() {
        cout << "Привет, меня зовут " << name << endl;
    }

protected:
    string name;
    int age;
};
```
Сейчас метод `sayHello()` всегда работает одинаково.
Но в реальной жизни разные «типы людей» могут здороваться по-разному.
Например:
- студент может сказать: «Я студент»
- водитель: «Я водитель»
Давайте создадим два класса-наследника:
```c++
class Student : public Person {
public:
    void sayHello() {
        cout << "Привет, я студент, меня зовут " << name << endl;
    }
};

class Driver : public Person {
public:
    void sayHello() {
        cout << "Привет, я водитель, меня зовут " << name << endl;
    }
};
```
Мы как будто «переопределили» поведение метода `sayHello()`.
Теперь используем их:
```c++
int main() {
    Student s;
    s.setName("Алекс");

    Driver d;
    d.setName("Иван");

    s.sayHello();
    d.sayHello();

    return 0;
}
```
Результат будет разный:
```plain text
Привет, я студент, меня зовут Алекс
Привет, я водитель, меня зовут Иван
```
Хотя метод называется одинаково — `sayHello()`.
Вот это и есть полиморфизм: **один интерфейс (метод), но разное поведение.**
Но здесь есть важный момент. Сейчас это просто «переопределение», но не полный полиморфизм. Или же **полиморфизм в упрощённом виде**.
## Задача
Создайте базовый класс `Person`, как у вас уже есть.
Затем создайте два класса-наследника:
- `Driver` (водитель)
- `Teacher` (учитель)
У каждого класса должен быть свой вариант метода `sayHello()`.
Также:
- у `Driver` добавьте метод `driveCar()`
- у `Teacher` добавьте метод `teach()`
---
### Что должно происходить в `main`:
1. Создать объект `Driver`
2. Задать имя и возраст
3. Вызвать `sayHello()` и `driveCar()`
4. Создать объект `Teacher`
5. Задать имя и возраст
6. Вызвать `sayHello()` и `teach()`
---
### Решение
```c++
#include <iostream>
using namespace std;

class Person {
public:
    void setName(string n) {
        if (n != "") {
            name = n;
        }
    }

    void setAge(int a) {
        if (a > 0) {
            age = a;
        }
    }

    void sayHello() {
        cout << "Привет, меня зовут " << name << endl;
    }

protected:
    string name;
    int age;
};

class Driver : public Person {
public:
    void sayHello() {
        cout << "Привет, я водитель, меня зовут " << name << endl;
    }

    void driveCar() {
        cout << name << " ведет машину" << endl;
    }
};

class Teacher : public Person {
public:
    void sayHello() {
        cout << "Привет, я учитель, меня зовут " << name << endl;
    }

    void teach() {
        cout << name << " учит учеников" << endl;
    }
};

int main() {
    Driver d;
    d.setName("Иван");
    d.setAge(30);

    d.sayHello();
    d.driveCar();

    cout << endl;

    Teacher t;
    t.setName("Мария");
    t.setAge(40);

    t.sayHello();
    t.teach();

    return 0;
}
```
