### Важно понять одну вещь
В C++ слово **«стек»** используется **в двух смыслах**:
1. **Стек как структура данных**
2. **Стек как область памяти программы**.
И вот здесь самое важное:
👉 **стек в памяти работает по тем же правилам, что и стек как структура данных**. Поэтому начнем с того что разберем стек как структуру данных. Так будет проще в дальнейшем понять как работает данная область памяти.
## **Стек как структура данных**
Структура данных в программировании — это способ организации данных для их эффективного использования. Вы уже видели несколько типов структур данных, например, массивы. Существует множество других структур данных, которые используются в программировании. Некоторые из них реализованы в Стандартной библиотеке C++, и стек как раз является одним из таковых.
Например, рассмотрим стопку (аналогия стеку) тарелок на столе. Поскольку каждая тарелка тяжелая, а они еще и сложены друг на друге, то вы можете сделать лишь что-то одно из следующего:
Посмотреть на поверхность первой тарелки (которая находится на самом верху).
Взять верхнюю тарелку из стопки (обнажая таким образом следующую тарелку, которая находится под верхней, если она вообще существует).
Положить новую тарелку поверх стопки (спрятав под ней самую верхнюю тарелку, если она вообще была).
В программировании стек представляет собой контейнер (как структуру данных), который содержит несколько переменных (подобно массиву). Однако, в то время как массив позволяет получить доступ и изменять элементы в любом порядке (так называемый ***«произвольный доступ»***), стек более ограничен.
**В стеке вы можете**:
Посмотреть на верхний элемент стека.
Вытянуть верхний элемент стека.
Добавить новый элемент поверх стека.
**Стек** — это структура данных типа **LIFO** (англ. *«**L**ast **I**n, **F**irst **O**ut»* = *«Последним пришел, первым ушел»*). Последний элемент, который находится на вершине стека, первым и уйдет из него. Если положить новую тарелку поверх других тарелок, то именно эту тарелку вы первой и возьмете. По мере того, как элементы помещаются в стек — стек растет, по мере того, как элементы удаляются из стека — стек уменьшается.
Мы только что разобрали **стек как структуру данных**.
## Что такое стек в памяти
**Стек памяти** — это специальная область оперативной памяти, которую программа использует для:
- вызовов функций;
- хранения **локальных переменных**;
- хранения параметров функций;
- хранения информации о том, *куда вернуться после выполнения функции*.
Проще говоря:
> 📦 Стек — это рабочий стол программы, где она временно хранит всё, что нужно прямо сейчас.
<empty-block/>
Для наглядности давайте представим всё графически. Детали здесь не так важны — главное понять принцип работы.
Представим, что перед нами вся **оперативная память компьютера**. Когда мы пишем программу, в том виде, в котором мы её компилируем, она уже заранее знает, сколько памяти ей потребуется. Мы объявляем переменные, массивы — особенно если они глобальные или статические. Даже если мы ещё не проходили статические массивы подробно, компилятор уже знает, сколько памяти они займут.
Когда программа загружается в память, под неё выделяется определённая область. Внутри этой области заранее резервируется место под все статические данные: переменные и массивы именно в том виде, в котором мы их объявили.
Примером статической памяти может служить объявление одномерного или двумерного массива. Когда мы объявляем статический массив, мы обязаны указать его размер — либо напрямую числом, либо перечислением элементов, чтобы компилятор сам смог вычислить его длину. Это и означает, что массив является **статическим**: программа заранее знает, сколько памяти он займёт.
Теперь поговорим про стек. Стек используется для хранения локальных переменных и информации о вызовах функций. Когда функция завершается, вся память, выделенная под неё в стеке, автоматически освобождается.<br>Однако у нас есть ещё одна возможность — **запросить дополнительную память у операционной системы во время работы программы**. Иногда мы просто не знаем заранее, сколько памяти нам потребуется. Именно эту проблему и решает **динамическая память**.<br>Представим, что кроме области памяти, занимаемой программой, у нас есть много свободной оперативной памяти. Если в процессе работы программы нам понадобится дополнительное место, мы можем попросить операционную систему (например, Windows) выделить нам ещё память. Если свободной памяти достаточно — она будет выделена, если нет — возникнет ошибка. (Но сейчас мы рассматриваем ситуацию когда у нас все хорошо)
В каких ситуациях это нужно? Например, при работе с базой данных. Вы можете не знать заранее, сколько записей вернёт запрос. Сегодня студентов может быть 100, завтра — 101, а послезавтра — 95. Кроме того, с базой данных могут работать другие пользователи, и результат запроса каждый раз может быть разным.
В такой ситуации невозможно заранее объявить массив фиксированного размера. Поэтому мы выделяем память **динамически**, в зависимости от того, сколько данных реально получили.
И вот здесь нам нужны **указатели и ссылки**.
В программе создаётся указатель — условно, это некий «ярлычок». Он занимает очень мало места и хранится в статической области программы. Сам указатель хранит **адрес в оперативной памяти**, где лежат реальные данные. Эти данные могут быть большими и находиться вне статической памяти программы.
Когда мы обращаемся к указателю, он «показывает», где именно в памяти лежит нужная нам информация, и мы можем с ней работать.<br>Эта область памяти, находящаяся вне программы и используемая для динамического выделения, называется **куча (heap)**. Такое название она получила потому, что туда можно выделять память в любом количестве и в любом порядке — пока не закончится оперативная память.

Важно помнить: если мы выделяем в куче массив, то его элементы всё равно должны располагаться **непрерывно**, потому что массив — это непрерывная область памяти. Хранить массив «кусочками» невозможно.
Основное предназначение указателей и ссылок — это **работа с динамической памятью**, то есть с кучей.
Теперь важный момент. Язык **C++** считается достаточно опасным с точки зрения работы с динамической памятью. Если вы выделили память в куче, вы **обязаны вручную освободить её**, когда она больше не нужна. Если этого не сделать, возникает **утечка памяти**.
Если программа постоянно выделяет память и не очищает её, рано или поздно у неё закончится оперативная память, и программа просто перестанет нормально работать.
В языках **Java** и **C#** такой проблемы нет. В них существует механизм **сборки мусора** — специальные алгоритмы отслеживают, используются ли объекты в программе. Если на объект больше нет ни одной ссылки, сборщик мусора автоматически освобождает память.
В C++ за этим нужно следить самостоятельно.
Это хорошо показывает разницу между **высокоуровневыми** и **низкоуровневыми** языками программирования. Более низкоуровневые языки (например, C++) дают более жёсткий и прямой контроль над памятью и ресурсами компьютера. Но за это приходится платить сложностью и большим количеством возможных ошибок.
Высокоуровневые языки проще и безопаснее, но они скрывают от программиста детали работы с памятью.
## А вот теперь мы можем перейти у указателям!
Указатель — это переменная, которая содержит адрес другой переменной.<br>Строго говоря, указатель может хранить адрес не только переменной, но и любого другого объекта в памяти. Однако на начальном этапе мы будем рассматривать указатель именно как переменную, содержащую адрес другой переменной.
## Что значит «адрес переменной»
Рассмотрим простой пример.
Раньше мы объявляли переменные так: `int a = 5;`<br>Если мы объявим ещё одну переменную и присвоим ей значение `a`: `int b = a;`<br>то переменная `b` получит **значение** переменной `a`, то есть число `5`.<br>Однако сама переменная `a` при этом тоже где-то **физически хранится в оперативной памяти**. У этой ячейки памяти есть **адрес**.
<columns>
	<column>
		Представьте оперативную память компьютера. Где-то в ней загружена наша программа, и в ней где-то лежит переменная `a`.
		В этой ячейке памяти хранится значение `5`, а сама ячейка имеет некоторый адрес. Конкретное число адреса нам не важно — важно лишь то, что **любой объект в программе имеет адрес в памяти**.
		Когда мы делаем обычное присваивание, мы копируем значение.<br>Но указатели работают иначе — они позволяют работать с адресом.

Разберём её по частям:
- `int` — тип переменной: указатель типа `int`
- `*px` — \* - говорит о том что у нас переменная является указателем + имя указателя
- `&a` — оператор взятия адреса (амперсанд)
Оператор `&` берёт **адрес переменной ****`a`**, и этот адрес сохраняется (присваивается) в переменной `px`.
	Таким образом, указатель `px` **указывает на область памяти**, где лежит переменная `a`. То есть px указывает на адрес переменной a.
Именно поэтому `px` и называется указателем.
Принято название указателей начинать с буквы p от слова pointer (\*pa, \*px, \*pnum и т.д.)
## Почему важно указывать тип указателя
Указатель **всегда имеет тип**, который должен соответствовать типу данных, на которые он указывает.
Например, если мы попытаемся написать так (пускай попробуют и почитают ошибку): `double* px = &A;`
где `A` имеет тип `int`, компилятор выдаст ошибку.
Он сообщит, что указатель на `int` нельзя использовать для инициализации указателя на `double`.
Это означает, что:
- указатель на `int` может указывать **только на ****`int`**
- указатель на `double` — только на `double`
- указатель на массив `int` — на массив `int` и так далее
Тип указателя всегда должен соответствовать типу данных, на которые он указывает.
## Что хранится в указателе на самом деле
Если мы просто выведем значение указателя `px` в консоль, мы увидим **адрес в памяти**, обычно в шестнадцатеричном виде.
Этот адрес может меняться при каждом запуске программы, потому что переменная может быть размещена в другой области памяти. Это нормально.
Главное — указатель хранит **адрес**, а не значение переменной.
```c++
int main() {
	
	int a = 5;
	
	int *px = &a;
	cout << px << endl;
	
	return 0;	
}
```
## Разыменование указателя
Теперь возникает вопрос:
**как получить значение, которое лежит по этому адресу?**
Для этого используется операция **разыменования**, которая обозначается символом `*`.
Если написать:
```c++
int main() {
	
	int a = 5;
	
	int *px = &a;
	cout << px << endl
	cout << *px << endl;
	
	return 0;	
}
```
то мы получим **значение**, хранящееся по адресу, на который указывает `px`.
В нашем случае это будет `5`.
Таким образом:
- `px` — адрес
- \*`px` — значение по этому адресу

## **Указатели и массивы**
На самом деле вы уже давно пользовались указателями , даже не осознавая этого — каждый раз, когда работали с массивами.
## Создание массива
Для примера создадим массив из пяти элементов и сразу инициализируем его значениями:
```c++
int main() {
	const int size = 5;
	int arr[size] = {4,55,79,12,8};
	
	for (int i = 0; i < size; i++) {
		cout << arr[i] << endl;
	}
}
```
Выведем массив на экран обычным способом.
Мы видим, что массив корректно хранит пять значений и мы можем к ним обращаться.
А теперь чудо и шок: **Имя массива — это указатель на его первый элемент.**
Это означает, что выражения:
```c++
int main() {
	const int size = 5;
	int arr[size] = {4,55,79,12,8}; // это
	
	for (int i = 0; i < size; i++) {
		cout << arr[i] << endl;
	}
	
	int *pArr = arr; // то же самое что и это
}
```
Обратите внимание на int \*pArr = arr;
мы **не используем оператор ****`&`**, потому что имя массива уже является указателем.
Теперь указатель `p` хранит адрес первого элемента массива `arr`.
То есть:
```c++
int main() {
	const int size = 5;
	int arr[size] = {4,55,79,12,8}; // это
	
	for (int i = 0; i < size; i++) {
		cout << arr[i] << endl;
	}
	
	cout << "---------------------------------------------------------" << endl;
	
	int *pArr = arr; // то же самое что и это
	for (int i = 0; i < size; i++) {
		cout << pArr[i] << endl;
	}
}
```
## Два способа вывода массива
Если написать два одинаковых цикла `for`:
- в одном использовать `arr[i]`
- в другом использовать `pArr[i]`
мы получим **одинаковый вывод**.
Это происходит потому, что:
- `arr` — указатель на первый элемент массива
- `pArr` — указатель, которому мы присвоили тот же самый адрес
Мы просто работаем с одной и той же областью памяти, но через разные имена.
## Проверка адресов памяти
Чтобы убедиться в этом, выведем значения `arr` и `p` в консоль:
```c++
cout << arr << endl;
cout << p << endl;
```
Мы увидим **одинаковые адреса в памяти**.
Это означает, что:
- имя массива
- указатель `p`
ссылаются на одно и то же место в оперативной памяти.
## **Передача параметров в функцию по указателю**
## Передача параметров по значению
Для начала окунемся в **область видимости и передачу параметров по значению**.
Создадим простую функцию:
```c++
void foo(int a) {
    a++;
}
```
В `main` объявим переменную и поработаем с ней:
```c++
int main() {
	int a =0;

  cout << a << endl; // первый вывод
	foo(a);
  cout << a << endl; // второй вывод
}
```
Результат будет таким:
```plain text
0
0
```
Почему так произошло?
- В функцию `foo` переменная передаётся **по значению**
- Значение `a` просто **копируется** в параметр функции
- Внутри функции мы работаем **с копией**
- После выхода из функции эта копия уничтожается
- Исходная переменная `a` в `main` не меняется
Даже несмотря на то, что внутри функции значение увеличилось, **снаружи это никак не отразилось**.
## Задача: изменить переменную извне функции
Теперь давайте поставим цель:
- **изменить переменную, объявленную в ****`main`****, изнутри функции**,
- **не возвращая значение через ****`return`**.
Для этого мы и будем использовать **указатели**.
## Напоминание: что такое указатель
Указатель — это переменная, которая хранит **адрес памяти**, где лежат данные.
Если мы:
- передадим в функцию **адрес переменной**,
- получим доступ к данным по этому адресу,
- изменим их,
то изменится **сама исходная переменная**, а не её копия.
## Передача аргумента по указателю
Изменим функцию `foo`:
```c++
void foo(int *pa) {
    (*pa)++; // разыменование
}

int main() {
	int a =0;

  cout << a << endl; // первый вывод
	foo(&a); // передаем адрес
  cout << a << endl; // второй вывод
}
```
Обратите внимание:
- функция принимает **указатель на ****`int`**
- чтобы изменить значение, мы используем **разыменование**
## Почему обязательно нужны скобки
Запись:
```c++
*pa++
```
❌ **неверна**, потому что:
- сначала выполнится `pa++`
- указатель сдвинется в памяти (арифметика указателей)
- и только потом произойдёт разыменование
Правильная запись:
```c++
(*pa)++;
```
✔️ Сначала разыменование
✔️ Потом инкремент значения

## Как вернуть два и более значения из функции
Обычно, когда мы объявляем функцию, мы указываем **тип возвращаемого значения**, и с помощью `return` функция возвращает **одно значение**.
Однако с использованием **указателей** мы можем сделать так, чтобы функция:
- не возвращала значение напрямую,
- но при этом **изменяла сразу несколько переменных**, переданных в неё.
Таким образом функция может «вернуть» **два, три и вообще любое количество значений** — в зависимости от задачи и, конечно же, от здравого смысла.
Пример с несколькими указателями
```c++
void foo(int *pa, int *px, int *pb) {
    (*pa)++; // разыменование
    (*px) = 67;
    (*pb) = -52;
}

int main() {
	int a = 0, x = 1, b = 5;

  cout << "a = " << a << endl; 
  cout << "x = " << x << endl;
  cout << "b = " << b << endl;
  
	foo(&a, &x, &b);
	
  cout << "a = " << a << endl; 
  cout << "x = " << x << endl;
  cout << "b = " << b << endl;
}
```
