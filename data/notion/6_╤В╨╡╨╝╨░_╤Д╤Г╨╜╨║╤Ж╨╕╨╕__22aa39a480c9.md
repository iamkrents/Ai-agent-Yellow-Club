## Зачем нужны функции
### Пример плохого кода
Представим, что нам нужно **три раза вывести строки «Привет!»** и **«Как дела?»**.
```c++
int main() {
	cout << "Привет!\n";
	cout << "Как дела?\n";
	
	cout << "Привет!\n";
	cout << "Как дела?\n";
	
	cout << "Привет!\n";
	cout << "Как дела?\n";
	
	return 0;
}
```
Минусы такого подхода:
- **Много повторений** одного и того же кода
- Если нужно что-то изменить (например, убрать восклицательный знак или добавить ещё одну фразу), придётся **исправлять код в нескольких местах**
- Код становится **длинным и неудобным**

## Что такое функция?
**Функция** — это:
> кусок кода с именем, который можно вызывать сколько угодно раз
Пример из жизни:
- Микроволновка — функция
- Кнопка «Пуск» — вызов функции
Более формально:
> **Функция** — это отдельный блок кода, который выполняет одну конкретную задачу.
Функции нужны для того, чтобы:
- Не писать один и тот же код много раз
- Делать программу **аккуратной и понятной**
- Разбивать большую задачу на маленькие и логичные части, в чем мы убедимся далее на примерах
По сути, функция — это **именованный блок кода**, который можно вызывать в разных местах программы.
## Формальное определение функции
Общий вид функции в C++:
```c++
тип имя_функции(параметры)
{
    тело функции
}
```
`Тип` - тип данных, который функция возвращает (например, `int`, `double` и все что изучали ранее). 
<callout icon="💡" color="gray_bg">
	Если функция не возвращает никакого значения, то используется тип **`void`**** (например если нам нужно просто что-то вывести, далее рассмотрим это на примере, чтобы было понятнее)**.
</callout>
`Имя_функции` - как мы будем её вызывать
<callout icon="💡" color="gray_bg">
	К именованию функции применяются те же правила, что и к именованию переменных
</callout>
`Параметры` - данные, которые функция принимает для работы. Функция может не иметь параметров, в этом случае указываются пустые скобки.
`Тело функции` — набор инструкций, которые выполняются при вызове функции
### Оператор return
Для возвращения результата функция применяет оператор **`return`**. Если функция имеет в качестве возвращаемого типа любой тип, кроме `void`, то она должна обязательно с помощью оператора `return` возвращать какое-либо значение
### Пример функции `main`
Например, определение функции `main`, которая должна быть в любой программе на языке C++ и с которой начинается ее выполнение:
```c++
int main()
{
    return 0;
}
```
Возвращаемым типом функции является тип **`int`**, поэтому функция должна использовать оператор **`return`** и возвращать какое-либо значение, которое соответствует типу `int`. Возвращаемое значение ставится после оператора `return`
<callout icon="💡" color="gray_bg">
	Стоит отметить, что С++ позволяет не использовать оператор `return` в функции `main`
</callout>
## Применяем функции к нашей задаче
Вернемся к нашей задаче. Зная, что такое функции и как их создавать, напишем функцию:
```c++
void sayHello() {
    cout << "Привет!\n";
    cout << "Как дела?\n";
}

int main()
{
    return 0;
}
```
Разберем функцию:
- `void` — тип, функция **ничего не возвращает**
- `sayHello` — имя функции
- `{}` — тело функции

## Выполнение и вызов функции
Когда запускается программа на C++, **автоматически запускается только функция ****`main`**.
Все остальные функции **не выполняются сами по себе** — их нужно **явно вызвать**.
Вызов функции осуществляется в форме:
```c++
	имя_функции(аргументы);
```
После имени функции указываются скобки. Если у функции нет параметров — скобки остаются пустыми.
### Применим это для нашей задачи:
```c++
void sayHello() {
    cout << "Привет!\n";
    cout << "Как дела?\n";
}

int main()
{
		sayHello();
		sayHello();
		sayHello();
		
    return 0;
}
```
<callout icon="🧑‍🎓" color="gray_bg">
	Что выведет программа? Почему это лучше, чем исходный код?
	<callout icon="🙋‍♂️" color="yellow_bg">
		Код стал короче и его проще редактировать
	</callout>
</callout>
Здесь определена функция sayHello, которая вызывается в функции main три раза. В этом и заключается преимущество функций: мы можем вынести некоторые общие действия в отдельную функцию и затем вызывать многократно в различных местах программы. В итоге программа три раза выведет строки "Привет!" “Как дела?”
## **Объявление функции**
При использовании функций стоит учитывать, что компилятор должен знать о функции до ее вызова. Поэтому вызов функции должен происходить после ее определения, как в случае выше. В некоторых языках это не имеет значение, но в языке C++ это играет большую роль. И если, к примеру, мы сначала вызовем, а потом определим функцию, то мы получим ошибку на этапе компиляции, как в следующем случае:
```c++
int main()
{
		sayHello();
		sayHello();
		sayHello();
		
    return 0;
}

void sayHello() {
    cout << "Привет!\n";
    cout << "Как дела?\n";
}
```
В этом случае перед вызовом функции надо ее дополнительно объявить. Объявление функции еще называют **прототипом**. Формальное объявление выглядит следующим образом:
```c++
тип имя_функции(параметры);
```
То есть для функции sayHello объявление будет выглядеть следующим образом:
```c++
void sayHello();

int main()
{
		sayHello();
		sayHello();
		sayHello();
		
    return 0;
}

void sayHello() {
    cout << "Привет!\n";
    cout << "Как дела?\n";
}
```
В данном случае несмотря на то, что определение функции идет после ее вызова, но так как функция уже объявлена до ее вызова, то компилятор будет знать о функции `sayHello`, и никаких проблем в работе программы не возникнет
## Функции с параметрами
### Зачем нужны параметры?
В предыдущем примере функция `sayHello` всегда выводила **одни и те же строки**.
Но что, если нам нужно выводить **разные данные**?
Например:
- приветствовать разных людей
- работать с разными числами
- выполнять одно и то же действие, но с разными значениями
Для этого функции умеют **принимать параметры**.
### Что такое параметры функции?
**Параметры** — это данные, которые передаются в функцию для её работы.
Можно сказать так:
> Параметры — это входные данные функции
### Синтаксис функции с параметрами:
```c++
тип имя_функции(тип1 параметр1, тип2 параметр2, ...) {
    // Тело функции, где используются параметр1, параметр2 и т.д.
}
```
- В скобках после имени функции перечисляются параметры: их тип и имя (как у переменных).
- Параметры могут быть любого типа (int, double, string и т.д.).
- Функция может иметь несколько параметров, разделенных запятыми.
- Если параметров нет — пустые скобки (как раньше).
### Пример функции с одним параметром
Усложним нашу задачу, которую писали ранее: нужно приветствовать разных людей, выводя "Привет, \[имя\]! Как дела?"
```c++
void sayHello(string name) {
    cout << "Привет, " << name << "!\n";
    cout << "Как дела?\n";
}
```
Разбор:
- `void sayHello(string name)` — функция `sayHello` принимает один параметр — строку `name`
- В теле функции `name` используется как переменная (выводится в cout). Данная переменная доступная **только внутри функции **(мы не можем использовать её в функции `main`)
Без параметров пришлось бы писать отдельные функции для каждого. С параметрами — одна функция.
### Вызов функции с параметром
```c++
имя_функции(аргумент1, аргумент2, ...);
```
- Аргументы — это конкретные значения (или переменные), которые передаются параметрам
- Количество и типы аргументов должны совпадать с параметрами функции
### Пример вызова функции с одним параметром
```c++

    cout << "Привет, " << name << "!\n";
    cout << "Как дела?\n";
}

int main()
{
		sayHello("Леша");
		sayHello("Паша");
		sayHello("Маша");
		
    return 0;
}
```

## Функция с несколькими параметрами
Функция может принимать **несколько параметров**.
**Задача:** Вычислить сумму двух чисел и вернуть результат. {color="green_bg"}
```c++
void sum(int a, int b) {
    cout << a + b << endl;
}

int main() {
    sum(3, 4);
    sum(10, 20);
    
    return 0;
}
```
<callout icon="💡" color="gray_bg">
	Порядок параметров имеет значение. Аргументы передаются **по порядку**, а не по имени.
</callout>
```c++
void subtract(int a, int b) {
    cout << a - b << endl;
}
```
Наглядно это можно рассмотреть на функции которая вычитает два числа:
```c++
subtract(70, 3); // 67
subtract(3, 70); // -67
```

## Возвращаемое значение функции. Оператор `return`
До этого мы рассматривали функции, которые **просто что-то выводят на экран**. Но часто функция должна не выводить результат, а **посчитать его и вернуть**, чтобы мы могли использовать его дальше в программе. Для этого используется оператор **`return`**.
### Что делает `return`?
Оператор `return` выполняет две важные задачи:
1. **Возвращает значение** из функции туда, где она была вызвана
2. **Завершает выполнение функции**
После `return` код в функции **больше не выполняется**.
Синтаксис: 
```c++
return выражение;
```
- где "выражение" должно соответствовать типу функции (например, для `int` — целое число или переменная типа `int`)
<callout icon="💡" color="gray_bg">
	Если функция имеет тип не `void`, она обязана содержать `return` (иначе ошибка компиляции). Можно иметь несколько `return` (например, в условных конструкциях), но после `return` функция завершается
</callout>
### Функция с возвращаемым значением
Если функция возвращает значение, её тип **не ****`void`**
***Пример:*** сумма двух чисел
```c++
int sum(int a, int b) {
    return a + b;
}
```
Разбор:
- `int` — функция возвращает целое число
- `return a + b;` — возвращаем результат вычисления
**Использование возвращаемого значения:**
```c++
int main() {
    int result = sum(5, 7); // 12
    cout << result + 40 << endl; // 52

    return 0;
}
```
Функция `sum`:
- получила аргументы `5` и `7`
- вычислила сумму
- вернула результат в переменную `result`
### Несколько `return` в функции
В функции может быть **несколько операторов ****`return`**, но выполнится **только один **— тот, до которого дойдёт выполнение. Это полезно в условных конструкциях (if-else), циклах или switch, где функция может завершиться в разных ветках кода с разными возвращаемыми значениями.
***Пример***: максимум из двух чисел
```c++
#include <iostream>
using namespace std;

int max(int a, int b) {
    if (a > b) {
        return a;  // Если a больше, возвращаем a и выходим
    }
    return b;      // Иначе возвращаем b
}

int main() {
    cout << "Максимум из 10 и 20: " << max(10, 20) << endl;
    cout << "Максимум из 5 и 3: " << max(5, 3) << endl;
    return 0;
}
```
Любой код **после ****`return`**** недостижим** и **никогда не выполнится**
```c++
int test(int x) {
    if (x > 0) {
        return 1;
        cout << "Положительное"; // никогда не выполнится
    }
    return 0;
}

```
<callout icon="💡" color="gray_bg">
	В `void`-функциях `return `можно использовать без значения для досрочного выхода: `return;`
</callout>
